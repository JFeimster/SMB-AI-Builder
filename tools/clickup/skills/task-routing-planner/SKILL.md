# Skill: Task Routing Planner

## Purpose

Plan task routing workflows for ClickUp or task-management systems so meetings, emails, intake forms, and CRM events become clear work items.

## Use When

Use this skill when a workflow needs tasks, owners, priorities, due dates, statuses, handoffs, or recurring checklists.

## Inputs Needed

- Source event
- Task type
- Owner role
- Priority rules
- Statuses
- Due date rules
- Review requirements

## Outputs

- Task routing map
- Status workflow
- Field checklist
- Automation candidates
- Human review points
- QA checklist

## Procedure

1. Identify task trigger.
2. Define the work item.
3. Assign owner logic.
4. Define statuses.
5. Define due date logic without inventing dates.
6. Add review checkpoints.
7. Create a pilot plan.

## Safety / Human Review

Do not invent commitments or deadlines. Sensitive customer, financial, legal, HR, lending, or compliance tasks require review before execution.

## Quality Checklist

- [ ] Trigger is clear.
- [ ] Task fields are defined.
- [ ] Owner logic is explicit.
- [ ] Human review is included.
- [ ] Pilot is small and testable.

## Example Prompt

```text
Design a task routing workflow for this process. Define triggers, task fields, owner logic, statuses, review checkpoints, and a safe pilot.
```

# SEO Operations Layer

## Purpose
This directory (`tools/seo/`) serves as the SEO operations layer for planning, generating, linking, reviewing, and refreshing the SMB AI Builder SEO page cluster. It ensures a consistent, ethical, and structured approach to organic growth and content management.

## Project Context
- **Project:** SMB AI Workflow & Agent Builder
- **Primary CTA URL:** https://chatgpt.com/g/g-6a3cac9b87f08191af3f98ab42ad55ae-smb-ai-workflow-agent-builder
- **SEO Positioning:** Workflow audit before automation. Score readiness before AI touches the business. Build safe SMB AI agent pilot plans.
- **Tone/Style:** Direct, anti-hype, operator-friendly, practical. No fake claims, no buzzwords.

## Files Overview
- `keyword-map.md`: Target keywords mapped to page types.
- `internal-link-map.md`: Strategy for internal linking across site clusters.
- `seo-page-template.md`: Blueprint for general SEO pages.
- `workflow-page-template.md`: Blueprint for workflow audit pages.
- `comparison-page-template.md`: Blueprint for balanced comparison pages.
- `metadata-rules.md`: Directives for metadata, OG, and titles.
- `schema-rules.md`: Approved structured data implementations.
- `sitemap-checklist.md`: QA steps for sitemap maintenance.
- `robots-llms-notes.md`: Guidelines for bot and LLM crawler files.
- `no-fake-schema.md`: Hard boundaries against black-hat SEO tactics.
- `seo-refresh-log.md`: Log for tracking content freshness and updates.
- `seo-qa-checklist.md`: Final review requirements before publishing SEO content.

# Copilot Review Checklist

When reviewing code or PRs, use this checklist to ensure compliance with project standards:

## 1. Content & Claims (CRITICAL)
- [ ] No fake proof, testimonials, ratings, or logos.
- [ ] No unsupported compliance or integration claims.
- [ ] Tone is direct and anti-hype. NO buzzwords (e.g., 'unlock your potential', 'revolutionize', 'seamless').
- [ ] For sensitive workflows (finance, HR, legal, compliance), the strict disclaimer must be visibly present near the CTA.

## 2. Links & CTAs
- [ ] All primary CTAs point EXACTLY to: `https://chatgpt.com/g/g-6a3cac9b87f08191af3f98ab42ad55ae-smb-ai-workflow-agent-builder`
- [ ] No relative paths for canonical URLs or OG tags (must use absolute paths pointing to `https://smb-ai-builder.vercel.app`).

## 3. Code Quality
- [ ] No secrets, API keys, or real environment variables.
- [ ] Static files are lightweight, using plain semantic HTML.
- [ ] Reuses existing `styles.css` and `script.js`. No new external frameworks unless explicitly approved.

## 4. SEO & Accessibility
- [ ] Exactly ONE `<h1>` per page.
- [ ] Valid JSON-LD schema present (Article/FAQPage).
- [ ] Adequate contrast, semantic tags, and visible focus states used.

## 5. Design
- [ ] Follows the neobrutalist, cyberpunk, futurist fintech aesthetic (high contrast, thick borders, hard offset shadows).

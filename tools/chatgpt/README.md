# ChatGPT Tool-Ops Layer

## Purpose
This directory (`tools/chatgpt/`) contains the ChatGPT tool-ops layer for the **SMB AI Workflow & Agent Builder** project. It is designed to act as a durable reference when using ChatGPT to generate, revise, QA, and expand the static site, SEO pages, partner embeds, prompt libraries, and documentation.

The SMB AI Workflow & Agent Builder helps small business owners, operators, freelancers, agencies, consultants, and automation builders audit workflows, score automation readiness, identify automation opportunities, design AI-powered workflows, and build safer AI agent pilot plans before buying tools, hiring developers, or automating broken processes.

## How to Use This Layer
Before prompting ChatGPT, provide it with the relevant context files from this directory to ensure the output adheres to the project's brand, tone, and technical requirements. Use this layer as a continuous guide to avoid drifting from core standards.

### Files to Reference Before Prompting
- `chatgpt-project-context.md` - Core project context, audience, and overall offer.
- `chatgpt-agent-profile.md` - Desired ChatGPT behavior and roles.
- `chatgpt-custom-gpt-notes.md` - Notes specific to the Custom GPT product, including what to avoid overclaiming.
- `chatgpt-copy-rules.md` - Copywriting rules (practical, direct, anti-hype, outcome-focused).
- `chatgpt-seo-rules.md` - Guidelines for generating SEO pages and schema.
- `chatgpt-page-generation-rules.md` - Rules for creating static HTML and Next.js pages.
- `chatgpt-embed-generation-rules.md` - Specifications for partner embed components.
- `chatgpt-prompt-library.md` - Reusable prompt templates for specific tasks.
- `chatgpt-output-formats.md` - Desired structure for final outputs (briefs, static files, packages).
- `chatgpt-conversation-starters.md` - Ready-to-use conversation starters for common tasks.
- `chatgpt-qa-checklist.md` - Checklist to run before finalizing any output.
- `chatgpt-changelog.md` - Log of updates and changes to this tool-ops layer.

## Core Directives
When generating content, keep the following hard rules in mind:
- **No hype:** Avoid vague AI buzzwords (e.g., "unlock your potential", "revolutionize", "seamless", "game-changing").
- **No fake proof:** Do not invent testimonials, ratings, revenue claims, user counts, logos, or case studies.
- **No risky advice:** Do not claim the GPT gives legal, financial, tax, HR, lending, accounting, or compliance advice. Always include disclaimers for sensitive workflows.
- **Brand style:** Use the established neobrutalist + futurist fintech + cyberpunk aesthetic in visual descriptions and tone.
- **Clear CTAs:** Keep calls to action obvious and linked directly to the Primary CTA URL.

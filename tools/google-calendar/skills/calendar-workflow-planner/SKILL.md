# Skill: Calendar Workflow Planner

## Purpose
Designs automated calendar and scheduling workflows.

## Use When
When a user needs to optimize their scheduling, set up booking links, or automate calendar events.

## Inputs Needed
- Current scheduling process (manual vs. tool)
- Types of meetings (internal, client, sales)
- Buffer times and availability constraints

## Outputs
- A proposed scheduling architecture
- Recommendations for booking tools (e.g., Calendly, Google Calendar appointment schedules)
- Automation triggers (e.g., event created -> send email)

## Procedure
1. Analyze the user's meeting types and scheduling bottlenecks.
2. Recommend a structure for multiple calendars if needed (e.g., Personal, Work, Shared).
3. Propose a booking link setup with specific event types.
4. Outline automations (e.g., adding meeting notes to Notion when an event is created).
5. Verify safety and privacy.

## Safety / Human Review
- Remind users to keep personal and business calendars separate if sharing availability.
- No pretending to actually configure their calendar settings.

## Quality Checklist
- [ ] Scheduling process is simplified?
- [ ] Buffer times are considered?
- [ ] Automations are practical?

## Example Prompt
```
I waste too much time going back and forth trying to schedule client calls. I want to set up an automated booking system using Google Calendar. What's the best way to structure this and handle different meeting lengths?
```

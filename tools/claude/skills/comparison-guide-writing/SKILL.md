# Comparison Guide Writing

## Purpose
Write balanced comparison content without attacking competitors.

## When to use this skill
Use when creating vs. pages (e.g., Make vs. Zapier, Custom vs. Off-the-shelf).

## When not to use this skill
Do not use to write biased, aggressive takedowns of other platforms.

## Inputs Claude should collect
- The tools or approaches being compared.
- The specific SMB use case.

## Files and folders this skill may touch
- `pages/seo/comparisons/`

## Source-of-truth files to read first
- `tools/jules/jules-seo-rules.md`
- `tools/jules/jules-copy-rules.md`

## Process
1. Frame the comparison as a decision guide based on readiness and workflow type.
2. Provide objective pros and cons for both sides.
3. Include a 'Use this when...' and 'Audit first when...' section.
4. Add a comparison table and the primary CTA.

## Output requirements
- Balanced, operator-friendly copy.
- Clear decision frameworks.

## Safety and claims rules
This GPT supports workflow planning and pilot design. It is not legal, financial, tax, HR, lending, accounting, medical, underwriting, or compliance advice. Sensitive decisions should stay under qualified human review.
Do not invent benchmarks, unsupported claims, or fake integrations.

## QA checklist
- Is the comparison fair and objective?
- Does it position the SMB AI Builder as a planning layer beforehand?
- Are there no attacks on competitors?

## Common mistakes to avoid
- Trashing a competitor to make a point.
- Inventing performance metrics.

## Final response format
HTML or Markdown file containing the comparison guide.

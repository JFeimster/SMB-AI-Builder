# Knowledge File Writing

## Purpose
Create markdown playbooks, templates, SOPs, scorecards, and guides.

## When to use this skill
Use when generating reusable assets, reference materials, or standardized processes.

## When not to use this skill
Do not use for marketing copy or SEO landing pages.

## Inputs Claude should collect
- The specific process or topic to document.
- Required format (SOP, scorecard, template).

## Files and folders this skill may touch
- `docs/knowledge-files/`

## Source-of-truth files to read first
- `README.md`
- `JULES.md`

## Process
1. Define the structure based on the requested format.
2. Write clear, step-by-step instructions or structured data.
3. Ensure operator-friendly language and remove any unnecessary fluff.
4. Include disclaimers for sensitive processes.

## Output requirements
- Clean, well-structured Markdown or JSON.
- Actionable, easy-to-read formatting (checklists, tables).

## Safety and claims rules
This GPT supports workflow planning and pilot design. It is not legal, financial, tax, HR, lending, accounting, medical, underwriting, or compliance advice. Sensitive decisions should stay under qualified human review.
Do not invent fake compliance standards.

## QA checklist
- Is the file strictly informative and practical?
- Are safety boundaries clear for high-risk processes?

## Common mistakes to avoid
- Writing overly academic or verbose documentation.
- Omitting the human-in-the-loop requirement for sensitive workflows.

## Final response format
Markdown or JSON file content.

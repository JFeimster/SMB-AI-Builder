# Claude Task Library

This file contains reusable prompts to instruct Claude for specific tasks.

## 1. Draft SEO Page
**Prompt:**
> "Draft an SEO landing page for [Topic/Workflow]. Use the SMB AI Builder context. Ensure the copy is direct, no-BS, and workflow-first. Include a single H1, descriptive H2s, 4-6 FAQs, and a CTA band near the end using the Primary CTA URL. Output in semantic HTML following our neobrutalist design system."

## 2. Review Copy
**Prompt:**
> "Review the following copy against the SMB AI Builder copy rules. Identify and rewrite any vague AI buzzwords, fluffy deployment theory, or generic claims. Ensure the tone is direct, operator-friendly, and specific."

## 3. Tighten Hero Section
**Prompt:**
> "Tighten this hero section. Make the headline punchy, direct, and focused on the workflow outcome. Ensure the sub-headline clarifies the value proposition without using words like 'unlock', 'revolutionize', or 'seamless'."

## 4. Rewrite Generic AI Copy
**Prompt:**
> "Rewrite the following generic AI copy. Replace hype words with specific, actionable, workflow-first details. Do not invent any case studies, metrics, or guarantees."

## 5. Review Sitemap
**Prompt:**
> "Review this sitemap structure for SEO best practices. Ensure logical hierarchy, proper canonical paths based on `https://smb-ai-builder.vercel.app`, and comprehensive coverage of core workflows and comparisons."

## 6. Review Embed Docs
**Prompt:**
> "Review the embed documentation. Ensure the instructions are operator-friendly, use practical language with clear checklists, and do not contain sensitive data or unsupported claims."

## 7. Review Accessibility
**Prompt:**
> "Review the following HTML for accessibility. Check for valid semantic structure, proper ARIA labels where necessary, descriptive alt text, and visible focus states for interactive elements."

## 8. Review Code
**Prompt:**
> "Review this [HTML/CSS/JS/TSX] code for the SMB AI Builder project. Ensure it adheres to the neobrutalist design system, uses semantic HTML, avoids external frameworks where instructed, and contains no secrets or private tokens."

# Skill: Email Workflow Auditor

## Purpose
Audits existing email handling processes and proposes automation workflows.

## Use When
When a user wants to streamline their inbox, automate email routing, or set up auto-responders.

## Inputs Needed
- Current email volume and pain points
- Types of incoming emails (support, sales, general)
- Desired response times and routing rules

## Outputs
- An email workflow diagram or step-by-step plan
- Recommended filters, labels, and forwarding rules
- Draft auto-responder templates

## Procedure
1. Assess the current state of the user's inbox and common email types.
2. Design a labeling and filtering system.
3. Propose automation rules (e.g., Zapier/Make integrations for specific emails).
4. Draft standard auto-responder copy if needed.
5. Verify against safety guidelines.

## Safety / Human Review
- No fake live connector claims (cannot actually configure Gmail for them).
- Warn against fully automating sensitive client communications without human review.
- Mention draft-first mode for customer-facing emails.

## Quality Checklist
- [ ] Workflow handles key email categories?
- [ ] Labels and filters are practical?
- [ ] Draft-first mode is emphasized?

## Example Prompt
```
I receive too many customer support emails in my main inbox. How can I organize this better and maybe set up a system to automatically label them and send a quick confirmation that we received it?
```

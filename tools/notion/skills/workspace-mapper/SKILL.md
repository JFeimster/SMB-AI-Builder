# Skill: Notion Workspace Mapper

## Purpose

Map Notion pages and databases into a practical SMB workflow system instead of a pile of pretty but useless pages.

## Use When

Use this skill when the workflow involves Notion pages, databases, dashboards, docs, CRM-lite setups, content calendars, or operating manuals.

## Inputs Needed

- Workspace/page purpose
- Existing page/database names
- Target audience
- Desired workflow
- Required fields/properties
- Current pain points

## Outputs

- Workspace map
- Database recommendations
- Property schema
- Utility gaps
- Page routing plan
- Human review notes

## Procedure

1. Identify the actual operating purpose.
2. Separate pages, databases, dashboards, and source docs.
3. Find duplicate or decorative pages.
4. Recommend database fields and views.
5. Map user actions to pages.
6. Add automation opportunities only after the workflow is clear.

## Safety / Human Review

Do not treat workspace access as permission to expose private records. Flag client, employee, finance, legal, or sensitive operating data.

## Quality Checklist

- [ ] Every page has a job.
- [ ] Databases have useful fields.
- [ ] Views match real workflows.
- [ ] Source docs and public pages are separated.
- [ ] Sensitive records are flagged.

## Example Prompt

```text
Map this Notion workspace into a useful operating system. Identify the real purpose, duplicate pages, missing database fields, priority views, and next workflow improvements.
```

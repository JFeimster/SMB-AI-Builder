# Skill: Notion Workspace Mapper

## Purpose
Designs the architecture and hierarchy of a Notion workspace.

## Use When
When setting up a new Notion workspace or restructuring an existing chaotic one.

## Inputs Needed
- Company size and structure (departments, teams)
- Primary use cases (project management, wiki, CRM)
- Current pain points

## Outputs
- A visual map or outline of the workspace structure
- Recommended root pages and sub-pages
- Permissions and sharing strategy

## Procedure
1. Assess the organization's structure and needs.
2. Design a top-level dashboard (e.g., 'Company Home').
3. Map out team-specific spaces and global databases.
4. Define a navigation strategy (e.g., linked databases, synced blocks).
5. Outline permission levels (Admin, Member, Guest).
6. Verify against safety rules.

## Safety / Human Review
- No fake live connector claims (cannot build the Notion pages directly).
- Advise on keeping sensitive data restricted.

## Quality Checklist
- [ ] Structure is scalable?
- [ ] Navigation is intuitive?
- [ ] Permissions strategy is included?

## Example Prompt
```
We are moving our small startup to Notion. We need a place for company policies, an engineering task board, and a sales CRM. How should we structure the workspace so it doesn't become a mess?
```

# Design System

This directory acts as the durable design-system layer for the SMB AI Builder site. It ensures visual consistency across landing pages, SEO pages, embeds, Next.js pages, and any other external-facing collateral.

## Design Direction

The overarching aesthetic is a blend of:
- Neobrutalist
- Futurist fintech
- Cyberpunk

The visual feel should evoke:
- A workflow war room
- A fintech risk dashboard
- A cyberpunk automation cockpit
- A loud startup launch page
- Practical SMB operations meets neon AI infrastructure

## Structure

*   **`brand-principles.md`**: Core brand values and tone.
*   **`design-tokens.md`**: CSS variables for colors, typography, spacing, etc.
*   **`color-system.md`**: Guidelines for applying the color palette.
*   **`typography-system.md`**: Rules for headings, body text, and labels.
*   **`layout-system.md`**: Grid, spacing, and container guidelines.
*   **`component-patterns.md`**: Specifications for reusable UI components.
*   **`css-class-map.md`**: A mapping of common CSS classes and their usage.
*   **`page-patterns.md`**: Layout structures for different types of pages.
*   **`motion-rules.md`**: Guidelines for animation and transitions.
*   **`accessibility-design-rules.md`**: Requirements for contrast, focus states, and readability.
*   **`visual-qa-checklist.md`**: A checklist to ensure designs meet the standard before shipping.
*   **`design-changelog.md`**: A log of updates and modifications to the design system.

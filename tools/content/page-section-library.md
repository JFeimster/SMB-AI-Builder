# Page Section Library

Use these predefined section blocks to structure landing and SEO pages.

## Hero
- **H1:** Punchy, anti-hype headline.
- **Subheadline:** Practical explanation of the value.
- **Primary CTA:** Directly to the GPT.

## Problem
- Highlight the risks of automating a broken process or skipping the planning phase. Emphasize wasted time and broken tools.

## Built for
- Identify the specific audience (e.g., Micro-SMBs, Agencies, Consultants).

## What it does
- Explain the mechanism: Audit -> Score -> Pilot Plan.

## How it works
- 1-2-3 step breakdown of the user journey with the GPT.

## Scorecard / Matrix
- Explain the 0-30 Readiness Scoring Model and what the bands mean (Do not automate yet, Clean up first, Pilot carefully, Ready).

## Blueprint
- Explain the output the user receives: a safe, actionable pilot plan.

## Use cases
- List 3-4 specific, practical examples of workflows that can be audited.

## Safety
- Highlight the importance of draft-only modes, human-in-the-loop, and include necessary disclaimers.

## CTA
- Final call to action band. Reiterate the value and provide the button.

## FAQ
- 4-6 relevant questions from the FAQ library.

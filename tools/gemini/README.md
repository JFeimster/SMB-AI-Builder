# Gemini Tool-Ops Layer

Welcome to the Gemini tool-ops layer for the SMB AI Workflow & Agent Builder project. This directory contains profiles, guidelines, and rules to standardize operations when using Gemini to review, audit, and critique repository assets.

## When to use Gemini

Gemini is expected to be used as a manual reviewer and critic. Use Gemini for:

- **Long-page review**: Analyzing long-form content, playbooks, and blueprints for consistency, tone, and accuracy.
- **SEO content review**: Evaluating pages for search intent, alignment, unique metadata, internal links, FAQ usefulness, and correct schema usage.
- **Competitive/positioning review**: Reviewing copy against competitors to ensure positioning is framed as a decision guide without attacking other tools.
- **Design screenshot critique**: Providing multimodal feedback on visual hierarchy, hero clarity, CTA visibility, contrast, and mobile layout.
- **Code review**: Inspecting HTML, CSS, JS, and TSX files for performance, correct semantic structure, and adherence to neobrutalist styling.
- **Accessibility review**: Auditing static pages for visual contrast, focus states, and overall web accessibility guidelines.

## Important Note on Automation

This layer does not assume any unsupported Gemini-specific automation. It acts as a reference library of instructions, rubrics, and structured guidelines to ensure consistent outputs when human operators collaborate with Gemini.

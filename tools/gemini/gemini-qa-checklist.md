# Gemini QA Checklist

Use this checklist to verify that output generated or reviewed by Gemini meets the repository's strict standards.

## 1. Tone & Copy
- [ ] Language is direct, practical, and operator-friendly.
- [ ] No vague AI buzzwords or "hype" phrases (e.g., "unlock your potential").
- [ ] No fake testimonials, ratings, reviews, logos, user counts, or case studies.
- [ ] Explicit disclaimers are present if the content touches on sensitive workflows (legal, financial, compliance, etc.).

## 2. Links & Assets
- [ ] The primary CTA points to the core GPT URL: `https://chatgpt.com/g/g-6a3cac9b87f08191af3f98ab42ad55ae-smb-ai-workflow-agent-builder`.
- [ ] Internal links are correctly formatted based on relative depth.
- [ ] Open Graph URLs and images point to absolute paths on `https://smb-ai-builder.vercel.app`.

## 3. SEO & Structure
- [ ] Exactly one `<h1>` per page.
- [ ] Unique meta titles and descriptions are provided for each page.
- [ ] JSON-LD schemas (Article, FAQPage) are valid and honest.
- [ ] Pages include a 4-6 item FAQ section.

## 4. Code & Visual Style
- [ ] Semantic HTML tags are used correctly.
- [ ] CSS aligns with the neobrutalist, cyberpunk, high-contrast aesthetic.
- [ ] No secrets, API keys, or real environment variables are present in the code or text.
- [ ] Responsive behavior (mobile-first) is maintained.

## 5. Automation Constraints
- [ ] The output does not assume automated, unverified scripts will be run by Gemini. It acts as an operator-assisted manual review.

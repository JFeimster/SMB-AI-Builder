# Codex Task Library

Use these as copy-ready task prompts inside Codex.

## Create a Static Site Page
Goal: Create or update a static product page for SMB AI Workflow & Agent Builder.

Context:
- Follow `AGENTS.md`.
- Reuse `styles.css` and `script.js`.
- Use the neobrutalist + futurist fintech + cyberpunk visual system.
- Keep the CTA pointed to `https://chatgpt.com/g/g-6a3cac9b87f08191af3f98ab42ad55ae-smb-ai-workflow-agent-builder`.

Constraints:
- Plain HTML only.
- No frameworks.
- No fake proof or unsupported claims.
- Include unique metadata.
- Include a CTA band and footer.

Done when:
- File exists in the requested folder.
- Relative CSS/JS paths are correct.
- Primary CTA works.
- Page is mobile-friendly and accessible.

## Create an SEO Page
Goal: Create a useful SEO guide page that targets the supplied keyword without becoming a thin doorway page.

Include:
- Unique title and meta description.
- One H1.
- Useful H2s.
- Internal links.
- FAQ section.
- Article JSON-LD and FAQ JSON-LD where appropriate.
- CTA to launch the GPT.

## Create a Workflow Audit Page
Use this exact format:

```txt
H1: Audit Your [Workflow] Before Automating It With AI

Sections:
- Why this workflow is tempting to automate
- What can go wrong
- What the GPT checks
- Readiness factors
- Suggested safe pilot
- Human review checkpoints
- CTA
```

## Create a Partner Embed
Goal: Create an iframe-safe partner embed.

Constraints:
- Max width: 940px.
- Max height: 1200px.
- Limited version of the site.
- Partner/affiliate use only.
- Primary CTA launches the GPT.
- Do not duplicate the whole landing page.

## Run a Conversion QA Pass
Check hero clarity, CTA visibility, audit-before-automation logic, hype removal, trust language, safety stance, and mobile layout.

## Run an Accessibility QA Pass
Check semantic landmarks, keyboard navigation, focus states, labels, contrast, FAQ accordion state, and reduced-motion support.

## Run a Deployment Readiness Pass
Check paths, missing files, sitemap, robots, metadata, Open Graph, JSON/XML validity, JavaScript errors, and hosting readiness.

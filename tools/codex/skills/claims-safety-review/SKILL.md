# Claims Safety Review

## Purpose
Review content for overreach, fake proof, unsafe claims, and missing sensitive-workflow boundaries.

## Use this skill when
- Auditing SEO pages, landing pages, comparison pages, workflow pages, or embeds.
- Checking claims before merge.
- Reviewing sensitive workflows involving money, finance, tax, accounting, legal, HR, medical, lending, underwriting, compliance, private data, or customer-facing decisions.

## Source-of-truth files to read first
- `tools/jules/JULES.md`
- `tools/jules/jules.json`
- `tools/jules/copy-rules.md`
- `tools/jules/qa-checklist.md`

## Process
1. Search for fake proof, fake reviews, fake ratings, fake logos, fake case studies, fake benchmarks, guarantees, and unsupported integrations.
2. Check that sensitive pages include the required disclaimer visibly near safety or CTA sections.
3. Replace hype with concrete workflow-planning language.
4. Return exact file paths and recommended edits.

## Required disclaimer
> This GPT supports workflow planning and pilot design. It is not legal, financial, tax, HR, lending, accounting, medical, underwriting, or compliance advice. Sensitive decisions should stay under qualified human review.

# Codex QA Checklist

## File QA
- [ ] Requested files were created or edited.
- [ ] No unrelated files were changed.
- [ ] File names and folders match the task exactly.
- [ ] Relative paths are correct.
- [ ] Links to root assets work.

## Conversion QA
- [ ] Hero is clear in five seconds.
- [ ] Primary CTA is visible above the fold.
- [ ] CTA language is punchy and specific.
- [ ] Page explains why users should audit before automating.
- [ ] Copy avoids generic AI claims.
- [ ] Final CTA repeats the main action.

## Trust QA
- [ ] No fake testimonials.
- [ ] No fake logos.
- [ ] No fake user counts.
- [ ] No fake review schema.
- [ ] No fake revenue claims.
- [ ] No unsupported compliance claims.
- [ ] Sensitive workflow disclaimer included where needed.

## SEO QA
- [ ] Unique title.
- [ ] Unique meta description.
- [ ] One H1.
- [ ] Useful H2s.
- [ ] Internal links included.
- [ ] FAQ section included where relevant.
- [ ] JSON-LD is valid when added.
- [ ] Page is not thin or duplicated.

## Accessibility QA
- [ ] Semantic landmarks.
- [ ] Good heading order.
- [ ] Accessible button/link text.
- [ ] Visible focus states.
- [ ] FAQ accordion exposes state.
- [ ] Color contrast is strong.
- [ ] Mobile layout stacks cleanly.

## Technical QA
- [ ] HTML validates reasonably.
- [ ] CSS does not create horizontal overflow.
- [ ] JavaScript does not throw obvious console errors.
- [ ] No heavy dependencies added.

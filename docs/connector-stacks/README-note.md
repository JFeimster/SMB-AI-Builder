# Connector Stack Implementation Note

Connector stacks are practical examples, not promises that integrations are live.

## Current Pattern

- Use `docs/connector-stacks/README.md` as the central index.
- Add one stack example per PR when possible.
- Keep every stack draft-first for sensitive workflows.

## File Naming

Use:

`docs/connector-stacks/{stack-name}.md`

Examples:

- `lead-follow-up-stack.md`
- `client-onboarding-stack.md`
- `sales-pipeline-stack.md`
- `repo-launch-stack.md`

## Required Sections

- What this stack is for
- Minimum stack
- Expanded stack
- Data flow
- What AI can draft
- What AI should not decide
- Risk level
- Setup checklist
- Starter GPT prompt
- Success criteria

# Connector Stacks

Welcome to the **Connector Stacks** documentation. This directory provides practical examples showing how SMB workflows can be planned safely with minimum viable tools.

## Minimum Viable Stack Philosophy

The goal of a connector stack is not to build a "fully autonomous business." Instead, the focus is on practical, step-by-step automation that a tired business owner can use without needing a systems engineering degree.

We emphasize the **minimum viable stack**: the fewest, simplest tools required to get a workflow running. Start small, verify that the core data flow works, and only expand with optional tools once the foundation is solid. Most importantly, always keep human review in the loop—AI should only draft safe outputs, while sensitive decisions and final approvals must remain with humans.

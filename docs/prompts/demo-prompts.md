# Demo Prompts

## Purpose
Prompts specifically designed to generate content for demos, presentations, or marketing the builder.

## When to use
Use when preparing a showcase of the SMB AI Builder's capabilities.

## Inputs needed
- Target audience for the demo
- Desired use case

## Demo workflow audit
**Prompt:**
```text
Generate a fictional but realistic workflow audit for a small plumbing business struggling with scheduling dispatch.
```
**Expected output:** A sample audit document highlighting pain points.
**Follow-up prompts:** Can you make the pain points more focused on lost revenue?
**Safety notes:** Do not use names of real local businesses.
**Human-review reminders:** Ensure the scenario is believable.
**Example use cases:** Creating slides for a webinar.
**Final QA checklist:**
- [ ] Scenario is realistic
- [ ] No real PII used

## Demo readiness scorecard
**Prompt:**
```text
Create a sample automation readiness scorecard for an e-commerce store with messy inventory data. Give them a score of 15/30 (Clean up first).
```
**Expected output:** A detailed, pedagogical scorecard showing why they scored poorly.
**Follow-up prompts:** Highlight the specific data cleanup steps needed.
**Safety notes:** Clearly mark as a hypothetical example.
**Human-review reminders:** Use to explain the scoring methodology to clients.
**Example use cases:** Client education materials.
**Final QA checklist:**
- [ ] Score matches prompt
- [ ] Explanations are clear

## Demo opportunity matrix
**Prompt:**
```text
Generate an 'Automation Opportunity Matrix' mapping 5 common HR tasks by Impact vs. Effort.
```
**Expected output:** A 2x2 matrix or formatted list ranking the tasks.
**Follow-up prompts:** Which task is the 'quick win'?
**Safety notes:** None.
**Human-review reminders:** Ensure the rankings logically make sense.
**Example use cases:** Sales collateral.
**Final QA checklist:**
- [ ] 5 tasks included
- [ ] Impact/Effort logic is sound

## Demo agent blueprint
**Prompt:**
```text
Draft a simplified AI Agent blueprint for a 'Website Chat Concierge' to show a prospective client what a safe design looks like.
```
**Expected output:** A clean, easy-to-read blueprint focusing on guardrails and handoffs.
**Follow-up prompts:** Emphasize the data privacy features.
**Safety notes:** Highlight the lack of access to backend systems in the demo.
**Human-review reminders:** Use to demonstrate safety-first design principles.
**Example use cases:** Proposal inclusion.
**Final QA checklist:**
- [ ] Easy to read
- [ ] Safety highlighted

## Demo pilot plan
**Prompt:**
```text
Create a 2-week pilot plan for testing a new automated invoice reminder system.
```
**Expected output:** A day-by-day or step-by-step pilot timeline.
**Follow-up prompts:** What are the success metrics for week 2?
**Safety notes:** Emphasize that the pilot runs in 'draft mode' only.
**Human-review reminders:** Good for showing structured implementation.
**Example use cases:** Closing a consulting deal.
**Final QA checklist:**
- [ ] Timeline is realistic
- [ ] Draft mode emphasized

## Demo static page content
**Prompt:**
```text
Write the hero copy and 3 feature highlights for a demo landing page showcasing the SMB AI Builder.
```
**Expected output:** Compelling, anti-hype marketing copy.
**Follow-up prompts:** Make it more operator-focused.
**Safety notes:** Adhere to anti-hype rules. No fake claims.
**Human-review reminders:** Check against brand voice guidelines.
**Example use cases:** Building the repo's public static site.
**Final QA checklist:**
- [ ] Anti-hype tone
- [ ] Operator-friendly

## Demo sales/resource page copy
**Prompt:**
```text
Draft a concise, pragmatic 'About' section explaining why we built the SMB AI Workflow Builder (focus on practical planning over shiny tools).
```
**Expected output:** Grounded copy explaining the philosophy.
**Follow-up prompts:** How does this save businesses money?
**Safety notes:** Avoid words like "revolutionize" or "game-changing".
**Human-review reminders:** Keep it direct and honest.
**Example use cases:** GitHub README or Notion site content.
**Final QA checklist:**
- [ ] Tone is pragmatic
- [ ] Forbidden words avoided

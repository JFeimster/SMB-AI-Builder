# Notion + Airtable Connector Playbook

## Connector

Notion and Airtable for workflow databases, operating pages, dashboards, and structured task/status records.

## Best For

- Workflow databases
- Client onboarding trackers
- Content calendars
- Internal ops dashboards
- CRM-lite systems
- Resource libraries

## Do Not Use For

- Final sensitive decisions
- Automating record changes with unclear ownership
- Replacing qualified review in finance, legal, HR, lending, underwriting, medical, or compliance workflows

## Required Inputs

- Workflow purpose
- Record types
- Required fields
- Statuses
- Users/owners
- Review requirements

## Common SMB Workflows

- Client onboarding
- Sales pipeline tracking
- Document request tracking
- Content production
- Project delivery
- Vendor/partner directories

## Automation Opportunities

- Draft database schema
- Suggest properties and views
- Generate status pipelines
- Create task routing maps
- Summarize stale records

## What Should Stay Human

- Final record approvals
- Sensitive status changes
- Customer-impacting decisions
- Financial or regulated conclusions

## Privacy / Security Risks

Databases often contain customer, operational, and financial data. Use least-privilege access and draft-first workflow design.

## Risk Level

Medium by default. High when records affect customers, money, regulated decisions, or private data.


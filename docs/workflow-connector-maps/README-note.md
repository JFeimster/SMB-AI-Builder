# Workflow Map Implementation Note

Workflow maps should be added incrementally.

## Current Pattern

- Use `docs/workflow-connector-maps/README.md` as the main index.
- Add one workflow map per PR when possible.
- Keep maps linked to connector playbooks that actually exist.

## File Naming

Use:

`docs/workflow-connector-maps/{workflow-name}.md`

Examples:

- `lead-follow-up.md`
- `client-onboarding.md`
- `sales-pipeline.md`
- `repo-to-deployment.md`

## Required Sections

- Workflow
- Common mess
- Best connectors
- Minimum stack
- Advanced stack
- What to automate
- What should stay human
- Risk level
- Draft-first rules
- Starter GPT prompt

# Jules Instructions for SMB AI Workflow & Agent Builder

## Project Overview
SMB AI Workflow & Agent Builder helps small business owners, operators, freelancers, agencies, consultants, and automation builders audit workflows, score automation readiness, identify automation opportunities, design AI-powered workflows, and build safe AI agent pilot plans before wasting money on tools, developers, or overbuilt automation systems.

This product is a planning and pilot-design assistant, not a replacement for professional judgment.

## Core Product Positioning
Stop automating broken workflows. Audit the mess first. Score readiness. Pick the safest pilot. Build an AI agent plan your business can survive.

## Static HTML Rules
- Use plain static HTML when creating static site pages.
- No external frameworks unless the repo already uses them.
- Reuse root `styles.css` and `script.js`.
- Use correct relative paths based on page location.
- Use semantic HTML.
- Use one H1 per page.
- Include accessible CTA links.
- Include visible focus states.
- Use mobile-first layouts.
- Do not bury CTA links.

## Next.js App Router Rules
- If working in `app/`, use Next.js App Router conventions.
- Keep components simple.
- Avoid unnecessary dependencies.
- Do not duplicate static pages unless specifically asked.
- Do not mix static HTML paths with App Router paths unless the repo intentionally supports both.

## SEO Rules
- Every SEO page needs a unique title.
- Every SEO page needs a unique meta description.
- Every SEO page needs one H1.
- Use descriptive H2s.
- Include internal links.
- Include a CTA band.
- Include FAQs where appropriate.
- Add JSON-LD where appropriate.
- Do not use fake review schema.
- Do not use fake ratings.
- Do not create thin doorway pages.
- Make pages genuinely useful and search-intent aligned.

## Accessibility Rules
- Use semantic landmarks.
- Use descriptive links.
- Use accessible button labels.
- Ensure keyboard navigation works.
- Ensure focus states are visible.
- Use sufficient contrast.
- Do not trap text in images.
- Add alt text for meaningful images.
- Hide decorative visuals from assistive tech where appropriate.
- Use ARIA only where needed.

## Safety and Compliance Boundaries
This GPT supports workflow planning and pilot design. It is not legal, financial, tax, HR, lending, accounting, medical, underwriting, or compliance advice. Sensitive decisions should stay under qualified human review.

For workflows involving customer-facing communication, money, finance, tax, accounting, legal, HR, medical, lending, underwriting, compliance, private data, or sensitive personal data, recommend:
- Draft-only mode
- Human approval before action
- Limited permissions
- Test data first
- Audit logs
- Escalation rules
- Rollback plan
- Qualified human review

Sensitive workflows require qualified human review.
